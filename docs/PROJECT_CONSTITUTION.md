# MarketDEX Project Constitution

**Version:** 1.0\
**Status:** Active

## Mission

MarketDEX is an offline-first Windows desktop application that helps
Pokémon TCG collectors and businesses make better decisions through
visual analytics, business intelligence, and streamlined workflows.

**Tagline:** *Every Collection Has a Story. Every Decision Has Data.*

## Permanent Principles

### Collector First

The software adapts to the user---not the other way around.

### Decision First

Every screen should help answer: - Should I Buy? - Should I Sell? -
Should I Grade? - Should I Hold? - Should I Trade? - Should I Open?

### Visual First

Every major module should begin with: 1. KPI Cards 2. Line Charts 3. Bar
Graphs 4. Pie Charts 5. Recommendations 6. Tables 7. Actions 8. Status
Strip

### Effortless First

-   Automatic saving
-   Workspace memory
-   Automatic backups
-   Offline-first
-   Minimal prompts

## Five Pillars

-   ❤️ Collection
-   💼 Business
-   🔬 Research
-   📊 Analytics
-   🧠 Intelligence

Every feature should strengthen at least one pillar.

## Architecture Rules

-   Build reusable engines before features.
-   Reusable widgets before page-specific components.
-   SQLite is the application's source of truth.
-   Dashboard-first user experience.
-   Charts should answer business questions.
-   Tables support investigation after insights.

## Engineering Rules

-   One responsibility per engine.
-   One responsibility per widget.
-   No duplicated code.
-   Reusable components whenever possible.
-   Document architectural decisions.
-   Test before merging.

## Definition of Success

A user should be able to open MarketDEX and, within 30 seconds,
understand: - Business health - Portfolio performance - Inventory
status - Opportunities - Risks - Recommended next actions

The software should work harder than the user.
