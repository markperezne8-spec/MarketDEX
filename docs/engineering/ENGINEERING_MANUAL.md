# ENGINEERING_MANUAL

## Purpose

This document defines the engineering standards for MarketDEX.

## Core Principles

-   Build reusable engines before features.
-   Keep modules independent.
-   UI never talks directly to the database.
-   Prefer composition over duplication.
-   One responsibility per engine and widget.

## Development Workflow

1.  Product Owner defines goals.
2.  Architect writes specification.
3.  Codex implements.
4.  Copilot reviews/refines.
5.  Test.
6.  Commit.
7.  Release.

## Required Deliverables

-   Architecture Brief
-   Codex Prompt
-   Copilot Prompt
-   Test Plan
-   Acceptance Checklist
-   Release Notes
-   Git Commit Message

## Definition of Done

-   Passes tests
-   Documentation updated
-   Architecture respected
-   Ready for GitHub
