# Release Candidate Acceptance

The checkpoint-contract milestone is accepted when:

- release authority is tied to all five permanent CI gates;
- the complete operator chain is named as the desktop regression surface;
- existing non-empty runtime data preservation is mandatory;
- the packaged Windows launch and relaunch checkpoint is explicit;
- Inventory → Pricing → Listing selection continuity is explicitly verified;
- listing planning remains separate from sale completion;
- operator authority over marketplace and sale outcomes is preserved; and
- the documentation does not claim a Windows verification result before the operator checkpoint is actually executed.
