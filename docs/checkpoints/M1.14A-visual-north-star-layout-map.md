## Purpose

This map translates the original MarketDEX Mission Control Visual North Star into safe, staged desktop implementation regions.

It is not a UI implementation. It does not authorize fake live data, action execution, marketplace integration, network access, polling, alerts, notifications, persistence, database migration, dependency changes, or business mutation.

This build preserves a strict no fake live data boundary.

## Build-order implications

M1.14B should focus on design tokens and reusable panel variants because most later regions need a shared visual language before data contracts.

M1.14C can safely introduce a header/status band shell only if unavailable evidence is labeled honestly and no live checks run.

M1.14D can safely improve the left navigation rail as long as routing behavior is preserved.

M1.14E and later should introduce dashboard section shells and contracts in small slices. Any section that implies real business intelligence must receive a dedicated immutable view-model or local service boundary before real values appear.

## Protected distinctions

- **Visual-only now** means styling, layout, shell labels, and honest unavailable states are allowed.
- **Contract-first** means no real-looking values, ranking, alerts, charts, trends, or business conclusions until a deterministic local evidence contract exists.
- **Mixed** means existing local values may be reused, but new North Star metrics still require explicit contracts.

## Acceptance gate

M1.14A is complete when this map exists in the repository and tests confirm that:

- the North Star source image exists;
- the required core regions are named in the map;
- visual-only and contract-first boundaries are preserved;
- no UI implementation files are changed by this build.
